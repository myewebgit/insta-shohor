How Does JavaScript Work?

JavaScript is a client-side scripting language and one of the most efficient, commonly used scripting languages, works with HTML and CSS to build web apps or web pages. It works with the web browser and with the Document Object Model (DOM). The web browser loads a web page, parses the HTML code and CSS styles, and creates the contents live and dynamic through DOM. Mostly every web browsers nowadays have their own JavaScript engines that understands the code and runs it. Our code is first received by the "parser" inside the JS engine. It checks the JavaScript code for syntactic errors, so whenever an error is detected by the parser, it shows error and stops execution. After that it creates Abstract Syntax Tree (ABT). Then it converts to machine code. When the program converted in the machine language (or in byte code), the converted code is sent to the system for execution, and finally the code run by the system/engine.

Difference between Local Storage and session storage.

1. Local Storage has expiration time, but session Storage will only be accessible while the data window is open. 2. Local Storage can be used for long term use. Beside this, Session Storage can be used when someone needs to store something that changes or something temporary. 3. Local Storage data stored for long time if the user manually clear the data then the data may be cleared but session storage data is cleared when the session or the browser is closed.
